# 3x-ui Automated Deployment Script

Автоматический скрипт быстрого развертывания панели управления **3x-ui** с интеграцией бесплатного SSL-сертификата от Certbot (Let's Encrypt), веб-сервером Nginx в качестве Reality-заглушки (Reality Fallback), проксированием панели/подписок через стандартный порт `443/tcp` и автонастройкой UFW.

Скрипт разработан на основе подробного руководства: [3x-ui гайд](https://noname-28.gitbook.io/3x-ui-gaid).

---

## Возможности скрипта

1. **Автоматическая проверка требований**: проверка прав суперпользователя (root) и совместимости ОС (Debian/Ubuntu).
2. **Параметризация и интерактивность**: скрипт можно запускать с аргументами командной строки для полностью автоматической настройки, либо в интерактивном режиме с пошаговыми подсказками в консоли.
3. **Безопасная настройка файрволла**: автонастройка `ufw` с гарантированным сохранением доступа к порту `22/tcp` (SSH), а также открытием портов `80`, `443`, порта панели и подписок.
4. **SSL от Let's Encrypt**: выпуск и привязка SSL сертификата к доменному имени.
5. **Nginx Reality Fallback & Reverse Proxy**:
   *   Настройка Nginx на работу через Unix-сокет (`unix:/dev/shm/nginx.sock`) с поддержкой PROXY протокола для маскировочного сайта.
   *   Настройка обратного проксирования (Reverse Proxy) для панели управления (`/путь_панели/`) и подписок (`/sub/`, `/clash/`, `/json/`).
   *   **Панель и подписки теперь доступны по защищенному HTTPS без необходимости указывать порты в адресе!**
6. **Сайт-заглушка**: скачивание и развертывание красивого SaaS-лендинга в качестве маскировки.
7. **Автоматическая установка 3x-ui**: тихая установка панели 3x-ui без необходимости интерактивно отвечать на вопросы инсталлятора, с последующим применением заданных параметров.

## Как запустить на сервере

### Вариант 1. Интерактивный запуск

Загрузите скрипт, сделайте его исполняемым и запустите. Скрипт сам поочередно спросит у вас все настройки:

```bash
wget -O deploy-3xui.sh https://raw.githubusercontent.com/sk4y3z/3huyadeploy/main/deploy-3xui.sh
chmod +x deploy-3xui.sh
sudo ./deploy-3xui.sh
```

---

### Вариант 2. Автоматический запуск с аргументами

Вы можете передать все параметры в виде аргументов, чтобы скрипт выполнился полностью без вашего участия:

```bash
sudo ./deploy-3xui.sh \
  -d yourdomain.com \
  -e admin@yourdomain.com \
  -p 2053 \
  -u admin \
  -w admin123 \
  -b panel \
  -s 2096
```

**Опции скрипта:**
*   `-d` | `--domain` — ваше зарегистрированное доменное имя, направленное на IP сервера (обязательно).
*   `-e` | `--email` — email для уведомлений о продлении сертификата Let's Encrypt (опционально).
*   `-p` | `--port` — внутренний порт панели (по умолчанию: `2053`).
*   `-u` | `--username` — имя пользователя панели управления (по умолчанию: `admin`).
*   `-w` | `--password` — пароль администратора панели (по умолчанию: `admin`).
*   `-b` | `--basepath` — секретный путь к панели управления (по умолчанию: `panel`).
*   `-s` | `--subport` — внутренний порт подписок (по умолчанию: `2096`).

## Результат работы скрипта

После успешной настройки ваши ссылки будут выглядеть следующим образом:
*   **Вход в панель управления:** `https://yourdomain.com/secretpath/` (где `secretpath` — ваш секретный путь)
*   **Ссылка для подписок в клиентах:** `https://yourdomain.com/sub/` (вместо неудобного `http://yourdomain.com:порт/`)

## Последующая настройка Reality в панели 3x-ui

Войдите в панель и добавьте подключение (Inbound):

1. Перейдите в раздел **Подключения (Inbounds)** -> **Добавить подключение (Add Inbound)**.
2. Заполните настройки следующим образом:
   *   **Протокол**: `vless`
   *   **Порт**: `443`
   *   **Reality**: Включите переключатель (Enabled)
3. В настройках Reality:
   *   **Dest**: `unix:/dev/shm/nginx.sock`
   *   **Server Names (SNI)**: введите ваш домен (например, `yourdomain.com`)
   *   **Fallback**: укажите тип `dest` как Unix-сокет и путь к сокету: `unix:/dev/shm/nginx.sock`
   *   **Ключи (Keys)**: нажмите кнопку **Get New Keys**, чтобы сгенерировать публичный и приватный ключи
4. Создайте или выберите клиента (параметр **Flow**: `xtls-rprx-vision`).
5. Нажмите кнопку **Добавить** для сохранения.
